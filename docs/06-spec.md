# AIFunc Package Format Spec

> **Target audience**: Package authors
> **Content**: Defines the file structure and field specifications for AIFunc packages. Organize files according to this spec to be recognized, compiled, and code-generated by the CLI
> **Spec version**: 0.1.0

---

## 1. What Is a Package

An AIFunc package is a directory that describes an AI function using declarative files:

- What input it accepts and what output it returns (`api.json`)
- What it is and what it does (`package.json`)
- How it guides the AI to complete the task (`prompts/general.md`)

A package **contains no executable code**.

---

## 2. Directory Structure

```text
<package-name>/
├── package.json         [Required] Package identity and metadata
├── api.json             [Required] Function input/output contract
├── prompts/
│   └── general.md       [Required] Prompt template
├── model-params.json    [Optional] Suggested model call parameters for the Engine
├── mock.json            [Optional] Static test data
├── README.md            [Recommended] Documentation
└── LICENSE              [Recommended] License
```

---

## 3. package.json

Package identity information and runtime configuration.

| Field | Type | Required | Description |
|:---|:---|:---:|:---|
| `name` | string | ✅ | Package name (lowercase letters, digits, hyphens, 2-64 characters) |
| `type` | string | ✅ | Fixed as `"standalone"` |
| `version` | string | ✅ | SemVer format, e.g., `"1.0.0"` |
| `description` | string | ✅ | Functionality description |
| `engine` | string | ✅ | Required AIFunc Engine compatible version, supports SemVer ranges (e.g., `"^0.1.0"`, `">=0.1.0"`) |
| `engineOptions` | object | | Engine runtime behavior configuration |
| `engineOptions.injectOutputSchema` | boolean | | Default `true`, see explanation below |
| `author` | object | | Author information |
| `author.name` | string | | Author name |
| `author.email` | string | | Contact email |
| `author.url` | string | | Personal or project homepage |

All sub-fields within the `author` object are optional, but if `author` is provided, at least `name` is recommended.

**`engineOptions.injectOutputSchema` explanation**:

- `true` (default): The Engine automatically appends output format directives to the prompt based on the `api.json` output schema. Package authors don't need to describe the return format in the prompt.
- `false`: The Engine does not inject format directives. Package authors must write complete output format guidance in `prompts/general.md` themselves.

**Example**:

```json
{
  "name": "@text-toolkit/summarizer",
  "type": "standalone",
  "version": "1.0.0",
  "description": "Generate a concise summary, returning the summary content and approximate word count.",
  "engine": "^0.1.0",
  "engineOptions": {
    "injectOutputSchema": true
  },
  "author": {
    "name": "Your Name",
    "email": "you@example.com",
    "url": "https://yoursite.com"
  }
}
```

Package names support `@scope/name` format (e.g., `@text-toolkit/summarizer`) for grouping related packages. Scope is purely a namespace and doesn't change the package's independence.

## 4. api.json

Defines the function's input and output types. The CLI uses this to generate strongly-typed code, and the Engine uses it for runtime validation.

| Field | Required | Description |
|:---|:---:|:---|
| `name` | ✅ | Function name (source for the generated code's function name). Lowercase letters, digits, underscores, 1-64 characters |
| `description` | ✅ | Functionality description (written into generated code comments) |
| `input` | ✅ | Input Schema, follows JSON Schema Draft 2020-12 |
| `output` | ✅ | Output Schema, follows JSON Schema Draft 2020-12 |

`input` and `output` use standard JSON Schema types: `string`, `number`, `integer`, `boolean`, `object`, `array`.

**Example**:

```json
{
  "name": "summarize",
  "description": "Generate a concise summary.",
  "input": {
    "type": "object",
    "properties": {
      "text": { "type": "string", "description": "Text to summarize" },
      "maxLength": { "type": "integer", "description": "Maximum word count for the summary", "default": 80 }
    },
    "required": ["text"]
  },
  "output": {
    "type": "object",
    "properties": {
      "summary": { "type": "string", "description": "The generated summary" },
      "wordCount": { "type": "integer", "description": "Approximate word count of the summary" }
    },
    "required": ["summary", "wordCount"]
  }
}
```

## 5. prompts/general.md

The instruction template for the AI. The Engine renders template variables at runtime and sends it as the System Prompt to the model.

**Template variables**:

| Syntax | Description |
|:---|:---|
| `{{input.fieldName}}` | Insert the specified field value from the input object |
| `{{input_json}}` | Serialize the entire input as a JSON string and insert |

When the input `type` in `api.json` is `string` (i.e., the entire input is a single string, not an object), use `{{input}}` to insert the raw text directly.

When input is an object, `{{input}}` is equivalent to `{{input_json}}`.

**Example**:

```markdown
# System

You are a concise and accurate summarization assistant.

## Task

Generate a summary for the user-provided text.

## Requirements

- The summary language must match the input text — do not translate.
- The summary should be concise, accurate, and fluent.
- Preserve the most important information; do not fabricate anything not in the original.
- `summary` must not exceed the word count specified by `maxLength`; default to 80 if not provided.
- `wordCount` should reflect the approximate word count of the summary.

## Input

Text to summarize: {{input.text}}

Maximum length: {{input.maxLength}}
```

If `engineOptions.injectOutputSchema` in `package.json` is `true` (default), you don't need to describe the output format in the prompt. If set to `false`, you need to specify the expected return JSON structure in the prompt yourself.

## 6. model-params.json (Optional)

Suggested model call parameters for the Engine. If this file is not provided, the Engine uses its own defaults.

**Priority** (lowest to highest):

1. Engine defaults
2. This file's configuration
3. Values explicitly passed in calling code

| Field | Type | Description |
|:---|:---|:---|
| `rules` | array | Parameter rule list |
| `rules[].match` | object | Match condition |
| `rules[].params.temperature` | number | Temperature |
| `rules[].params.topP` | number | Sampling top-p value |
| `rules[].params.maxTokens` | integer | Maximum output token count |

**match conditions** (choose one of three):

| Field | Type | Description |
|:---|:---|:---|
| `match.model` | string | Exact match on model name |
| `match.models` | string[] | Match any model name in the list |
| `match.pattern` | string | Regex match on model name |

**Example**:

```json
{
  "rules": [
    {
      "match": { "pattern": ".*" },
      "params": { "temperature": 0.1, "maxTokens": 500 }
    }
  ]
}
```

## 7. Naming Rules Summary

| Constraint | Rule | Example |
|:---|:---|:---|
| Character encoding | All text files UTF-8, no BOM | — |
| Package name | Lowercase letters, digits, hyphens, 2-64 characters | `summarizer` |
| Scoped package name | `@scope/name` format | `@text-toolkit/summarizer` |
| Function name | Lowercase letters, digits, underscores, 1-64 characters | `summarize` |
| Version | SemVer 2.0.0 | `1.0.0` |

## 8. Complete Examples

A minimal complete package (required files only):

```text
summarizer/
├── package.json
├── api.json
└── prompts/
    └── general.md
```

**package.json**:

```json
{
  "name": "summarizer",
  "type": "standalone",
  "version": "1.0.0",
  "description": "Generate a concise summary, returning the summary content and approximate word count.",
  "engine": "^0.1.0",
  "author": { "name": "Your Name" }
}
```

**api.json**:

```json
{
  "name": "summarize",
  "description": "Generate a concise summary.",
  "input": {
    "type": "object",
    "properties": {
      "text": { "type": "string", "description": "Text to summarize" },
      "maxLength": { "type": "integer", "description": "Maximum word count for the summary", "default": 80 }
    },
    "required": ["text"]
  },
  "output": {
    "type": "object",
    "properties": {
      "summary": { "type": "string" },
      "wordCount": { "type": "integer" }
    },
    "required": ["summary", "wordCount"]
  }
}
```

**prompts/general.md**:

```markdown
# System

You are a concise and accurate summarization assistant.

## Task

Generate a summary for the user-provided text.

## Requirements

- The summary language must match the input text — do not translate.
- The summary should be concise, accurate, and fluent.
- Preserve the most important information; do not fabricate anything not in the original.

## Input

Text to summarize: {{input.text}}

Maximum length: {{input.maxLength}}
```
