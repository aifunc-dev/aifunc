# AIFunc 路线图

我们的目标是让 AI 能力在所有主流编程语言中成为标准化的、强类型的、可预测的。以下是我们的方向。

---

## 当前（开发中）

正在推进的功能，预计近期可用。

- [x] **Go 语言支持**：一等公民的 Go 代码生成。
- [x] **Java 支持**：将 AI 函数引入企业 JVM 生态。
- [x] **C# 支持**：强类型 C# 包裹代码生成（.NET 6+，零 NuGet 依赖）。
- [ ] **健壮的结构化输出处理**：对复杂结构化响应进行全面的解析和校验。
- [ ] **录制/回放 Mock**：自动将真实 API 响应录制为 mock 文件，实现零配置的确定性测试。
- [ ] **模型特定参数适配**：逐步添加强类型的、提供商特定的参数支持，具备完整的 IDE 自动补全。

> **设计哲学：协议封装，调用方可覆盖**
> 模型特定参数（如 `temperature`、`top_p`）已由包作者在 AIFunc 协议中**预定义并优化**。作为调用方，你通常不需要配置任何东西——包会根据你使用的模型自动选择合适的参数。但如果你通过 config 对象显式覆盖，IDE 会根据 `model` 字符串提供精确的、提供商特定的自动补全。
>
> **致模型提供商**：我们欢迎合作。如果你希望 AIFunc 支持你的模型的完整参数集，请提 issue 或直接联系我们。

---

## 接下来（已规划）

已确认的下一批优先级功能。

- [ ] **流式输出**：优雅的、强类型的异步迭代器，支持流式响应。
- [ ] **CLI 包搜索**：`aifn search` 轻松发现社区包。
- [ ] **CLI 第三方插件系统**：可扩展的 CLI 插件架构。
- [ ] **Rust 核心**：目标为 Protocol v1.0。Rust 将作为高性能基础库，为其他语言（如 Swift、Ruby、Dart）提供绑定——而非替代上述的一等公民 Java/Kotlin、Go 或 C# 实现。
- [ ] **官方包注册中心与管理平台**：一个集中式平台，用于发现、发布、版本管理和管理社区驱动的 AI 包，提供完整的生命周期支持。

---

## 明确不做

为保持「AI 即函数」的哲学，AIFunc **不会**原生构建或支持以下功能。

- **状态与存储（如向量数据库、原生 RAG）**
  AIFunc 严格无状态。我们不管理内存或数据库。如果你需要 RAG，在你的业务逻辑中检索上下文，然后作为普通参数传给 AIFunc。

- **智能体编排与规划**
  我们不会构建 DAG 运行器、自主规划或复杂的 Agent 循环。编排不需要新框架；用标准的 `if`、`switch`、`for` 循环编排你的 AI 函数即可。
